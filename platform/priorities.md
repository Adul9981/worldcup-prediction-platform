# 建设优先级

更新时间：2026-06-14。

## 当前最高优先级

### 1. 激活真正线上后台

目标：后台保存的今日推荐能被所有访客看到。

对应待办：

- A-001：连接 Vercel KV 或 Upstash Redis 存储。
- A-002：复验真正线上今日推荐后台。
- C-001：真正线上今日推荐后台。

验收：

- Vercel 项目有 `KV_REST_API_URL` 和 `KV_REST_API_TOKEN`。
- 后台保存成功后，固定域名前台、无痕窗口、另一浏览器都能看到同一内容。
- 保存失败时后台明确提示失败，不假装发布成功。

### 2. 建立日常维护流程

目标：用户每天能稳定维护今日推荐，不依赖 Codex 临时记忆。

对应待办：

- A-003：建立后台运维小手册。
- C-003：发布前检查清单。

验收：

- 有固定后台地址、管理口令使用方式、保存后检查步骤。
- 有发布/排错 checklist。

### 3. 继续打磨数据底层，不自动填充今日推荐

目标：底层事件库和状态判断更准，但前台今日推荐仍只使用人工录入。

对应待办：

- B-001：新增/消失选题检测。
- B-002：更完整的机会评分。
- B-003：捕获 Polymarket `neg_risk` 官方字段。
- B-004：比赛结束后的事件状态同步。

验收：

- 事件库能解释市场状态变化。
- 比赛结束后相关事件能进入结束或复核状态。
- 低风险、高风险、每日关注等重点清单不自动乱填。

## 中优先级

### 4. 前台体验复验

目标：线上页面稳定、轻量、移动端可读。

对应待办：

- C-002：线上 API 错误提示优化。
- C-004：移动端视觉复验。

验收：

- 今日页、预测事件库、进度纵览在移动端不重叠。
- API 错误提示短而清楚。

- 今日推荐只来自后台手动录入。
- 宁可空着，不自动乱填。
- 用户可见层不输出仓位、下注金额或下注单位。
- Polymarket 外链必须带邀请码或联盟参数。
- 比赛状态默认使用北京时间。

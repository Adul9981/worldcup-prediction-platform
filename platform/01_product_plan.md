# 产品规划

## 产品名称

暂定：World Cup Prediction Desk

中文定位：世界杯预测市场交易工作台。

## 用户

第一阶段核心用户是我们自己，用来验证功能是否正常：

- 每天看比赛。
- 参与预测市场。
- 需要知道今天哪些事件值得关注。
- 需要避免情绪下注。
- 需要把判断过程复盘成内容。

长期目标是做成订阅付费产品。第一版先不做付费墙和账号系统，先验证数据、评分、仓位、复盘、内容输出是否可靠。

## 主要使用场景

### 场景 1：早上打开工作台

用户想知道：

- 今天有哪些比赛。
- Polymarket 当前有哪些世界杯相关市场。
- 哪些事件交易最活跃。
- 哪些事件被忽视。
- 哪些事件可能有高盈亏比。
- 今天最多能用多少仓位。

### 场景 2：赛前 90 分钟

用户想知道：

- 首发是否影响原判断。
- 哪些事件从观察升级为可参与。
- 哪些事件必须取消。
- 是否有价格突然变化。

### 场景 3：比赛中

用户想知道：

- 是否触发临场加仓。
- 是否需要减仓或停止。
- 盘口变化是否和比赛内容一致。

### 场景 4：赛后

用户想知道：

- 今天判断对在哪里。
- 错在哪里。
- 输赢是过程问题还是结果噪音。
- 明天仓位是否需要调整。
- 哪些内容可以发成推文。

## MVP 功能

### P0 必须有

- Polymarket 世界杯市场发现。
- 当前可参与事件数量。
- 市场分类：冠军、特殊事件、单场、球员、组合、未知。
- 今日赛事列表。
- 双轨事件评分：稳健机会、高盈亏比观察。
- 事件评分：关注度、忽视度、盈亏比、风险。
- 仓位建议。
- 待下注 / 等待 / 禁止下注三栏。
- 复盘记录。
- 每日文案输出。
- Polymarket 跳转链接统一带联盟参数：`?via=serene77mc-g6kj`。

### P1 很快要有

- SQLite 入库。
- 仪表盘页面。
- 事件价格历史快照。
- 冠军盘概率对比。
- 球队分层和市场价格关联。
- 每日 Markdown 简报自动生成。

### P2 后续增强

- WebSocket 实时价格。
- 多平台接口占位启用。
- 纸上交易曲线。
- 预测模型校准。
- 内容发布排期。
- 订阅付费方案设计。
- 分享页 / 公开内容页。

### 暂不做

- 真实交易执行。
- 自动下单。
- 钱包连接。
- 用户注册系统。
- 复杂权限系统。
- 第一版付费系统。

## 信息架构

第一屏必须是工作台，不做营销首页。

页面结构：

1. 今日工作台。
2. Polymarket 市场。
3. 赛事雷达。
4. 仓位账本。
5. 复盘库。
6. 文案库。
7. 数据源和设置。
8. 订阅产品规划页。后续再做，不进第一版核心流程。

## 核心指标

- 每天发现事件数量。
- 可参与事件数量。
- 被标记为高价值的事件数量。
- 实际参与事件数量。
- 纸上交易盈亏。
- 真实下注盈亏。
- 纪律错误次数。
- 复盘完成率。
- 内容产出数量。
- 联盟链接点击数量。后续接入。
- 订阅转化。后续接入。

# 世界杯预测市场交易工具平台章程

## 核心原则

本平台先规划，后建设。任何功能在进入开发前，必须先回答：

1. 解决什么决策问题。
2. 使用什么数据。
3. 输出什么行动建议。
4. 风险边界是什么。
5. 如何复盘它是否有用。

## 平台目标

构建一个世界杯预测市场交易工具平台，服务三件事：

1. 找到值得参与的预测事件。
2. 用纪律化仓位管理参与机会。
3. 将每日预测、下注逻辑、结果复盘转化成内容资产。

## 当前优先级

1. Polymarket 只读市场发现。
2. 世界杯赛事和球队数据库。
3. 每日事件雷达。
4. 仓位建议和风险限制。
5. 赛后复盘。
6. 内容文案系统。
7. 前端工作台。

## 当前明确不做

第一阶段不做：

- 真实下单。
- 钱包连接。
- 私钥管理。
- API key 存储。
- 自动交易。
- 桥接。
- gasless 交易。
- 承诺收益的展示或文案。

如果未来要做交易执行，必须单独通过规划、风险审查、纸上交易验证和用户确认。

## 平台角色

### 1. 数据层

负责采集和整理：

- 官方赛程。
- 球队和小组。
- Polymarket 市场。
- 盘口价格快照。
- 比赛结果。
- 复盘记录。

### 2. 分析层

负责判断：

- 哪些事件值得关注。
- 哪些事件被市场忽视。
- 哪些事件盈亏比最高。
- 哪些市场价格过热。
- 哪些事件必须等首发或临场。

### 3. 决策层

负责输出：

- 下注 / 不下注。
- 等待信息。
- 仓位建议。
- 加仓条件。
- 减仓或取消条件。

### 4. 内容层

负责生成：

- 每日赛前预测。
- 首发后更新。
- 临场机会。
- 赛后复盘。
- 社交平台分享推文。

### 5. 展示层

负责呈现：

- 今日工作台。
- 市场事件榜。
- 仓位和风险。
- 复盘与内容。
- 数据来源和更新时间。

## 决策准则

- 没有数据来源，不进入核心评分。
- 没有取消条件，不给仓位建议。
- 没有复盘字段，不允许记录为有效策略。
- 不能解释价格优势，只能标记为观察。
- 数据脏或来源不明，必须显示低置信度。
- 内容可以吸引人，但不能替代交易纪律。

## 成功标准

第一阶段成功标准：

- 每天能知道 Polymarket 上有多少世界杯事件。
- 每天能看到最值得关注、最被忽视、最大盈亏比事件。
- 每个事件都有数据来源、分类、价格、风险和行动建议。
- 每次下注或纸上交易都有记录和复盘。
- 每天能生成一份可分享的复盘/预测文案。

## 工作方式

后续每次建设必须按这个顺序：

1. 先更新规划或待办。
2. 明确验收标准。
3. 再写代码或改界面。
4. 运行验证。
5. 把新问题写入优化库。
6. 把重要原则写入原则标准。

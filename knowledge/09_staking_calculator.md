# 仓位计算器

## 目标

仓位计算器不是下注机器人，而是把预测事件评分转换成可执行的风险纪律。

当前版本只读运行，不连接钱包、不读取私钥、不调用交易端点。它的输出用于网站工作台展示、每日决策记录和复盘。

## 输入

- `market_event_scores`: 事件评分结果。
- `prediction_market_topics`: 市场选题和状态。
- `polymarket_champion_team_map`: 冠军盘球队映射。

## 输出

- `data/polymarket/latest_staking_recommendations.csv`
- `data/polymarket/latest_staking_recommendations_summary.md`
- SQLite 表：`staking_recommendations`

## 仓位单位

当前采用单位制，而不是金额制。

- `0`: 不下注，只观察或等待。
- `0.25`: 观察仓，高盈亏比或轻量验证。
- `0.5`: 小仓，适合稳健观察或高盈亏比小仓。
- `1.0`: 稳健小仓上限，冠军长期盘目前较少触发。
- `1.5` / `2.0`: 保留给未来短周期、高流动性、强确定性事件。

## 当前硬约束

- 每日总推荐仓位上限：`5.0` 单位。
- 单一市场上限：`2.0` 单位。
- 冠军 / 长期盘单市场上限：`1.0` 单位。
- 高盈亏比轨道默认不重仓。
- 风险评分过高时强制 `0` 仓位。

## 日额度规则

评分模型会保留所有候选机会，但仓位计算器只给日额度内的机会实际仓位。

如果某个事件通过基础仓位规则，但当天 5 单位额度已经分配完，则标记为：

- `日额度等待`

这类事件不是无价值，而是候选池。后续只有在价格变化、新闻变化、已持仓减仓或更高优先级事件消失后，才重新进入实际下注额度。

## 当前排序原则

优先顺序：

1. 稳健轨道。
2. 高盈亏比轨道。
3. 观察名单。
4. 中性等待。
5. 排除项。

同一轨道内按评分从高到低排序。

## 网站展示口径

网站不要把所有候选事件都显示为“建议下注”。

建议分成三层：

- 今日实际仓位：`recommended_stake_units > 0`
- 候选但等待额度：`action_label = 日额度等待`
- 只观察 / 等待 / 风险过高：`recommended_stake_units = 0`

## 重要边界

交易量低不等于机会低。低交易量事件可能只是刚上线、距离比赛还远、或当前市场尚未传播。

只有市场关闭、结算、规则失效或事件确定结束，才进入结束状态。否则应保留观察状态，并等待后续交易量和价格变化。

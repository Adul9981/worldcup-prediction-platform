# 世界杯赛事预测平台蓝图

## 平台目标

每天自动或半自动输出：

1. 今日比赛雷达。
2. 最值得关注事件。
3. 最被忽视事件。
4. 最大盈亏比事件。
5. 下注建议和仓位上限。
6. 首发后更新。
7. 比赛后复盘和分享文案。

## 模块 1：赛事数据库

数据表：

- teams
- groups
- matches
- event_markets
- odds_snapshots
- bet_ledger
- daily_reviews
- content_posts

核心字段：

- 队伍强弱等级。
- 是否明显弱势。
- 是否明显强队。
- 摇摆等级。
- 关注热度。
- 信息差等级。
- 小组出线动机。
- 当前仓位暴露。

## 模块 2：事件评分引擎

每个事件输出：

- 重要度分。
- 被忽视分。
- 盈亏比分。
- 风险分。
- 仓位建议。
- 取消条件。

评分公式初稿：

```text
event_score = importance * 0.25
            + ignored_value * 0.25
            + price_edge * 0.25
            + motivation_clarity * 0.15
            - risk_level * 0.10
```

## 模块 3：每日工作台

首页展示：

- 今日全部比赛。
- A/B/C 优先级。
- 最值得关注 Top 3。
- 最被忽视 Top 3。
- 最大盈亏比 Top 3。
- 今日已用仓位。
- 今日剩余仓位。
- 等首发列表。
- 禁止下注列表。

## 模块 4：仓位管理系统

输入：

- 总预算。
- 今日剩余额度。
- 事件评分。
- 相关盘口暴露。
- 连续盈亏状态。

输出：

- 0 单位：不下注。
- 0.25 单位：观察/试探。
- 0.5 单位：普通机会。
- 1 单位：清晰机会。
- 1.5 单位：高置信且价格好。
- 2 单位：极少数核心机会。

## 模块 5：文案系统

每天生成：

- 赛前预测帖。
- 首发更新帖。
- 临场机会帖。
- 赛后复盘帖。
- 每日总结长文。

原则：

- 内容吸引人，下注克制。
- 输出观点，但不承诺结果。
- 把“为什么等待”也写成内容。

## 模块 6：复盘系统

每个事件复盘：

- 判断是否正确。
- 价格是否正确。
- 入场是否纪律。
- 仓位是否合理。
- 结果是否噪音。
- 下次要改什么。

复盘标签：

- `good-process`
- `bad-price`
- `lineup-miss`
- `motivation-miss`
- `market-overheat`
- `lucky-win`
- `unlucky-loss`
- `discipline-error`

## 工具建设顺序

1. CSV/Markdown 知识库。
2. SQLite 数据库。
3. 每日 Markdown 简报生成器。
4. 仓位计算器。
5. 事件评分脚本。
6. 静态网页仪表盘。
7. 内容文案生成器。
8. 比赛后复盘分析器。
